New applications should start with net.java.truelicense.json or net.java.truelicense.core, depending on whether they want to use the Version-2-with-JSON (V2/JSON) or the Version-2-with-XML (V2/XML) license key format. The V2/JSON format produces smaller license keys. Creating V2/JSON format license keys is also faster than creating V2/XML format license keys (there is no notable difference on the license consumer side). However, using V2/JSON format license keys requires the Jackson JSON Processor to be present on the class path at runtime.

Applications which need to interoperate with license keys which have been generated by TrueLicense 1 need to use the Version-1 (V1) format in net.java.truelicense.core.
