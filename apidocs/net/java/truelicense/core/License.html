Class License

  • java.lang.Object
    • net.java.truelicense.core.License
  • Direct Known Subclasses:
    LicenseContent

    @ParametersAreNullableByDefault
@Nullable
public class License
extends Object
    A Java Bean which defines and provides the common properties of any license. All properties are set to null by default. However, license validation may fail if some properties are null when creating, installing or verifying license keys.

    There are two options for extending a license with custom properties:

    • The easiest way is to put the custom properties in a map or another JavaBean and store it in the extra property (composition).
    • Another option is to subclass this class and add the custom properties (inheritance). However, you may then need to override equals(Object), too. If you do, you also need to override hashCode() for consistency.
    Either way, the custom properties must support the Codec of the respective license key format - see CodecProvider.codec(). This is easiest to achieve if the respective class follows the JavaBean pattern just like this class does.

    Note that this class deviates from the JavaBeans specification in that it neither implements Serializable nor Externalizable. This is because object serialization is not required with the used Codecs and creates a major obligation for any subclass.

        public final int getConsumerAmount()
        Returns the amount of consumers which are allowed to use the licensing subject. The default value is one in order to retain compatibility with V1 format license keys and to reduce the size of the XML encoded form (default values don't need to get encoded).

      • getConsumerType

      • getExtra

        public final Object getExtra()
        Returns the license extra data. Unlike the other properties of this class, this property is supposed to hold private information which should never be shared with the customer. It is typically used by applications in order to provide some extra data for custom validation.

      • getHolder

      • setTerm

        public final void setTerm(int days)
        Conveniently computes the issued date/time, notBefore date/time and notAfter date/time properties from the given number of days.

        Note that this computation depends on the system clock unless the property issued is set. This is not a security issue because this method is neither used for license initialization nor validation by a license consumer manager.

        Parameters:
        days - the validity period in (24 hour) days from now.
